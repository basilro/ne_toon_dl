📖 네이버웹툰 다운로드 — 사용 매뉴얼

기본은 무료 회차 자동 다운로드입니다. 설정 → 쿠키/경로 → 작품 등록 → 스케줄 실행 순서로 따라 하세요.

  2. 네이버 쿠키 준비하기

1. 빠른 시작 (5단계)

  1. 쿠키 주입 — 설정 > 일반의 네이버 쿠키에 로그인 쿠키(JSON)를 붙여넣습니다. (→ 2번)
  2. 다운로드 경로 지정 — 다운로드 경로에서 저장할 폴더를 선택합니다.
  3. 작품 등록 — 체크할 웹툰에 작품을 한 줄에 하나씩 입력 (제목 / URL / titleId 숫자 모두 가능).
  4. 설정 저장 → 쿠키 검증 — 상단 설정 저장 후 쿠키 검증으로 로그인 상태를 확인합니다.
  5. 실행 — 즉시 실행으로 바로 받거나, 설정 > 자동 탭에서 스케줄러를 등록합니다. 진행 상태는 진행 상황 탭에서 확인합니다.
처음엔 작품 1개 + 즉시 실행으로 테스트한 뒤, 잘 받아지면 스케줄을 켜는 것을 권장합니다.

2. 네이버 쿠키 준비하기

이 플러그인은 사용자의 네이버 로그인 세션(쿠키)으로 네이버웹툰에 접근합니다. 비밀번호는 저장하지 않습니다.

  1. 크롬에서 https://comic.naver.com 에 로그인합니다.
  2. 크롬 확장 Cookie-Editor를 설치합니다.
  3. 네이버웹툰(또는 naver.com) 탭에서 Cookie-Editor를 열고 Export → JSON 으로 복사합니다.
  4. 복사한 JSON을 설정 > 일반 > 네이버 쿠키 칸에 붙여넣고 설정 저장.
  5. 쿠키 검증 버튼으로 유효 여부를 확인합니다.
필수 쿠키는 NID_AUT(네이버 통합 로그인)입니다. 쿠키는 만료되면 다시 export해서 주입하세요. 웹훅을 설정해두면 쿠키 만료 시 1회 알림을 받습니다. (→ 9번)

3. 설정 항목 상세

일반 탭
항목설명
체크할 웹툰다운로드 대상. 한 줄에 하나씩(또는 |). 제목 / 작품 URL / titleId 숫자 모두 인식.
예) 신의 탑 · https://comic.naver.com/webtoon/list?titleId=183559 · 183559
네이버 쿠키Cookie-Editor JSON. (→ 2번)
다운로드 경로저장 최상위 폴더. 네이버웹툰 이미지는 원본 jpg 그대로 저장(변환 없음).
회차 폴더 ZIP 압축On이면 정상 완료된 회차 폴더를 NNNN_회차.zip으로 묶고 원본 폴더 삭제. 부분 실패 회차는 압축하지 않습니다. (→ 6번)
공지 기반 자동 다운이번 달 유료화 전환 공지에서 작품을 추출해 자동 다운로드. (→ 4번)
유료화 작품 저장 폴더명공지 다운분이 들어갈 하위 폴더 이름. 기본 완결.
프록시 사용 / URL외부 프록시 경유. (→ 9번)
이 플러그인은 기본적으로 무료 회차만 받습니다. 유료/잠금 회차는 스킵됩니다.
알림 탭
  • 쿠키 만료 알림 URL — 만료 감지 시 1회 발송.
  • 다운로드 완료 알림 URL — 자동 실행 후 받은 회차가 있을 때 요약 발송.

Discord 웹훅 기본, Slack 웹훅도 자동 인식. URL 옆 테스트 버튼으로 수신을 먼저 확인하세요.

자동 탭
  • 스케쥴링 실행 정보 — Interval(분) 또는 Cron. 예) 180 또는 0 */3 * * *.
  • 시작시 자동실행 — On이면 SJVA 시작 시 스케줄러 자동 등록.

4. 유료화 공지 자동 다운로드

공지 기반 자동 다운을 켜면, 자동 실행 시 네이버웹툰 공지의 “YYYY년 M월 유료화 전환 작품 안내”를 찾아 작품을 추출하고 무료 회차를 받습니다.

  • 매일 1회 공지만 체크(API 1회) — 새 공지가 없으면 즉시 종료합니다. 추천 스케줄: 매일 09시 (0 9 * * *).
  • 서비스가 꺼졌다 켜져도 다음날 09시에 자동으로 따라잡습니다(catch-up).
  • 이번 달 공지만 처리하며, 전환일(M월 N일)이 이미 지났으면 자동 스킵합니다.
  • 저장 위치는 다운로드경로/완결/... (폴더명은 유료화 작품 저장 폴더명).
  • 설정 상단 공지 즉시 실행 버튼으로 이번 달 공지를 지금 바로 처리할 수도 있습니다(토글 상태와 무관).

5. 수동 다운로드

수동 다운로드 탭은 작품 하나를 즉석에서 골라 받을 때 씁니다.

  1. 작품 제목 또는 작품/회차 URL(또는 titleId)을 입력하고 분석.
  2. 회차 목록에서 받을 회차를 체크합니다(무료·미수신 회차만 선택 가능).
  3. 선택 다운로드로 직렬 다운로드. 취소로 중단.
수동 다운로드는 직접 누르는 동작이라 자동 다운로드와 독립적입니다. 단, 압축 등 다른 작업이 돌고 있으면 충돌 방지를 위해 끝난 뒤 다시 시도하라는 안내가 뜹니다.

6. 저장 구조와 중복 처리

폴더 구조
다운로드경로/
└─ 작품제목/
   ├─ info.xml / cover.jpg
   ├─ 0001_회차제목/      (압축 Off)
   │   ├─ 001.jpg ...
   └─ 0002_회차제목.zip   (압축 On — 폴더 대신 zip)

다운로드경로/완결/작품제목/...   ← 유료화 공지 기반 다운분

회차 폴더 이름은 NNNN_회차제목(4자리 0채움)이며, 회차는 내부적으로 회차번호(no)로 식별합니다.

이미 받은 회차는 어떻게 판단하나?
  • 기본 판단 기준은 DB의 다운로드 이력(회차 단위 completed)입니다. 이력에 완료로 있으면 다시 받지 않습니다.
  • 압축(ZIP) 사용 시: DB 이력에 없더라도 다운로드 폴더에 해당 회차 zip이 이미 있으면 다시 받지 않고 이력에만 인식(등록)합니다. 회차번호(no) 기준으로 찾으므로 회차 제목이 조금 달라도 매칭됩니다.
  • zip은 정상 완료된 회차에만 만들어지므로 “zip 있음 = 온전히 받은 회차”로 안전하게 간주합니다. 받다 만 폴더(zip 없음)는 다시 받습니다.
  • 압축 처리 중 이미 같은 이름의 zip이 있으면 덮어쓰지 않고 기존 zip을 유지합니다.
처음 도입해서 “예전에 받아둔 zip”을 이력에 인식시키고 싶을 때, 압축을 켠 상태로 자동/수동 실행을 한 번 돌리면 해당 작품의 기존 zip들이 재다운로드 없이 이력에 등록됩니다.
압축을 끄고 폴더로 보관하는 경우엔 이 “디스크 인식”이 동작하지 않습니다(완전 다운 여부를 zip 으로만 보장하므로). 중복을 디스크로 방지하려면 압축을 켜는 것을 권장합니다.

7. 도구 버튼 (설정 탭 상단)

버튼동작
설정 저장변경한 설정을 저장.
즉시 실행스케줄을 기다리지 않고 자동 다운로드를 지금 1회 실행.
쿠키 검증현재 쿠키로 로그인 상태인지 확인.
공지 즉시 실행이번 달 유료화 공지를 지금 처리(설정 토글과 무관).
메타 동기화작품 폴더에 info.xml/cover.jpg가 없으면 생성. watchlist가 비어도 다운로드 폴더를 스캔해 대상을 찾습니다.
압축 처리다운로드 폴더 아래 모든 회차 폴더를 .zip으로 묶고 원본 폴더 삭제. 이미 zip인 회차는 건너뜀.
이력 초기화다운로드 이력 DB를 정리(파일은 삭제하지 않음).
메타 동기화 · 압축 처리 · 공지 즉시 실행은 백그라운드로 돌며, 진행 상황은 진행 상황 탭에서 확인합니다. 자동 다운로드가 실행 중이면 충돌 방지를 위해 거부됩니다.

8. 진행 상황 · 다운로드 이력

  • 진행 상황 — 자동/수동 다운로드의 현재 상태와 작품·회차·페이지 진행률을 실시간으로 표시.
  • 다운로드 이력 — 받은 회차 목록. 상태 필터 · 제목 검색을 지원하고 선택 삭제가 가능합니다.

9. 알림(웹훅) · 프록시

웹훅 알림
  • Discord 웹훅 URL 예: https://discord.com/api/webhooks/{ID}/{TOKEN} (Slack 웹훅도 자동 인식).
  • 쿠키 만료 / 다운로드 완료 알림을 각각 켤 수 있습니다. (설정 > 알림)
프록시 (선택)
  • 프록시 사용 On + URL 값이 있을 때만 적용. 예) http://user:pass@host:8080 · socks5h://host:1080.
  • On이면 API 호출·이미지 CDN 다운로드가 같은 프록시를 사용합니다.
  • 트래픽 부담이 커지므로 불안정하면 끄세요.

10. 문제 해결 (FAQ)

받아지는 게 0개예요 / “쿠키 만료” 가 떠요

쿠키 만료 가능성이 큽니다. 네이버웹툰에 다시 로그인 → Cookie-Editor로 재-export → 쿠키 칸에 붙여넣고 설정 저장 → 쿠키 검증으로 확인하세요.

유료/잠금 회차는 안 받아지나요?

네. 이 플러그인은 무료 회차만 받습니다. 유료/잠금 회차는 스킵됩니다.

같은 회차를 자꾸 다시 받아요

압축(ZIP)을 켜두면 디스크의 기존 zip을 회차번호로 인식해 재다운로드를 피합니다. (→ 6번) 폴더 보관(압축 Off) 상태에서는 디스크 인식이 동작하지 않습니다.

공지 자동 다운이 안 도는 것 같아요

공지는 하루 1회 체크라 즉시 반영되지 않을 수 있습니다. 지금 확인하려면 공지 즉시 실행을 누르고 로그 탭을 확인하세요. 이번 달 공지가 없거나 이미 처리됐으면 아무것도 받지 않습니다.

스케줄이 안 도는 것 같아요

설정 > 자동에서 스케줄러 등록·실행 상태를 확인하고 로그 탭에서 실행 로그를 확인하세요. 즉시 실행으로 먼저 동작을 점검할 수 있습니다.

※ 이 플러그인은 본인이 정당하게 접근 권한을 가진 콘텐츠를 개인 보관 목적으로 내려받는 용도입니다. 네이버 이용약관과 저작권법을 준수해 사용하세요.
